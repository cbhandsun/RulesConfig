# WMS 策略编排中心：高级配置与业务指南

本指南旨在帮助业务专家和实施工程师理解 **WMS 策略编排中心** 的核心逻辑及高级功能。系统通过非线性逻辑、全局红线和影子模拟，提供了工业级的策略配置能力。

---

## 1. 策略管线流转模式 (Pipeline Flow Modes)

在策略管线（规则层级）中，系统支持三种核心流转逻辑，用于定义规则间的“衔接”关系。

| 模式 | 视觉标识 | 业务语义 | 使用场景 |
| :--- | :--- | :--- | :--- |
| **瀑布降级 (Waterfall)** | 橙色实线 | 逐级尝试。只有当前规则找不到可行解时，才执行下一条规则。 | 常规货位寻址，从最优库区向次优库区降级。 |
| **维度累加 (Pipeline)** | 绿色虚线 | 逻辑增强。前一规则的结果将作为后一规则的输入，过滤和加权持续叠加。 | 多维评分，例如：[库位距离] + [库位热度] + [库存周转] 的综合评分。 |
| **条件跳转 (Jump)** | 紫色箭头 | 逻辑越迁。满足特定匹配谓词后，直接跳过中间环节执行目标规则。 | 紧急/特种业务流程。如：VIP 订单直接跳转至“快速出库”规则。 |

---

## 2. 算子执行算子逻辑 (Step Execution Relationships)

在单个规则（Rule）内部，执行节点（Steps）决定了具体的物理寻址或管控行为。你可以通过点击步骤间的 **逻辑连接器** 来切换模式：

- **业务维度累加 (Join/Merge)**: 节点间的逻辑是“且”。即所有节点共同作用于候选集合，进行多重过滤和加权计算。
- **满足即止 / 寻址失败降级 (Fallback)**: 节点间的逻辑是“备选”。如果第一个算子锁定了结果，后续算子不再执行；仅在前一算子寻址失败时，启用“备选方案”。

---

## 3. 全局策略红线 (Global Guardrails)

**全局红线** 是横跨所有策略的物理约束层。

- **核心价值**: 解决跨策略的共同禁令（例如：冷链区严禁存放常温商品、维保区域库位禁用）。
- **执行优先级**: 全局红线拥有 **最高优先级**。无论具体规则如何配置，凡是触发全局红线的操作都会被强制阻断。
- **独立管理**: 可以在“全局独立规则库”中预设，并一键挂载到指定的场景策略中。

---

## 4. 影子模式 (Shadow Mode)

在发布新策略前，利用 **影子模式** 进行无损验证。

- **Baseline 对比**: 系统自动抓取当前的“生产策略”作为基准（Baseline），与新编辑的“影子策略”进行对比。
- **历史数据回演 (Historic Replay)**: 模拟器会提取真实的历史订单指纹，模拟在不同策略下的执行结果（如：总路径、寻址成功率、拣货时长）。
- **蒙特卡洛模拟**: 对策略进行压力测试，预测在订单波峰期间策略的稳健性。

## 5. 典型业务场景案例

### 5.1 复杂分支寻址 (Complex Branching)
- **场景**: 根据货品体积（大件/小件）自动分流。
- **配置**: 
  - 规则 1：匹配货品体积 > 1m³，执行 `Jump -> 大件区规则`。
  - 规则 2：匹配常规体积，执行 `Waterfall -> 标准区规则`。

### 5.2 多维动态评分 (Multi-dimensional Scoring)
- **场景**: 拣货路径最短且同批次聚合度最高。
- **配置**: 
  - 规则模式切换为 `Pipeline (累加)`。
  - 步骤 A：应用 `Distance Factor (距离因子)`。
  - 步骤 B：应用 `Batch Aggregation (批次聚合因子)`。

### 5.3 紧急降级策略 (Emergency Fallback)
- **场景**: 自动化立库故障时，自动切换至手动阁楼货架。
- **配置**: 
  - 规则 1：尝试 `AS/RS (自动化寻址)`，步骤设为 `Fallback`。
  - 如果 API 报错或库存不足，自动滑入下一规则 `Manual Zone (人工寻址)`。

## 6. 节点参数与算子上下文 (Node Parameters & Step Context)

在算子（Step）层面，**节点参数** 是实现精细化管控的“控制按钮”。

### 6.1 参数的确定机制 (Subject-Driven)
节点参数并非固定，而是由该算子的 **“寻址主体 (Target Subject)”** 动态决定的：
- **库位型 (LOCATION)**: 参数包含 `maxDistance` (最大动线距离)、`capacityBuffer` (库位预留周转容积) 等。
- **库存型 (INVENTORY)**: 参数包含 `fifoWeight` (先进先出权重)、`minExpiryDays` (效期硬阈值) 等。
- **资源/任务型 (RESOURCE/TASK)**: 参数包含 `loadLimit` (负载上限)、`snReq` (序列号管控级别) 等。

### 6.2 参数与步骤的关系
- **属性从属**: 参数是步骤的“私有属性”。同一策略中，即使两个步骤使用了相同的计算因子，通过配置不同的节点参数，也能实现完全不同的业务边界。
- **分类治理**: 节点参数按业务逻辑深度分为三类：
  1. **约束类 (Constraint)**: 筛选时用，决定准入资格（如：溢收阈值）。
  2. **调节类 (Adjustment)**: 评分时用，决定优选顺序（如：FIFO 权重）。
  3. **行为类 (Behavioral)**: 决策后用，决定处理路径（如：异常自动挂起）。
- **代码注入**: 在定义脚本因子（Script/SQL）时，可以直接引用这些参数（如 `params['maxDistance']`），实现业务逻辑与底层代码的解耦。

## 7. 决策流控机制 (Flow Control)

本系统采用递归式流控设计，规则 (Rule) 与阶段 (Step) 共享同一套逻辑机制：

### 7.1 流转模式定义
1. **继续流转 (CONTINUE)**:
   - **语义**: 维度累加。
   - **行为**: 将当前计算出的候选集或评分传递给下一层级，进行进一步的“过滤”或“叠加分”。
2. **满足即止 (TERMINATE)**:
   - **语义**: 顺延降级。
   - **行为**: 一旦当前层级找到了符合要求的结果（如分配到了库位），立即停止后续逻辑并直接输出结果。
3. **条件跳转 (JUMP)**:
   - **语义**: 非线性路由。
   - **行为**: 根据特定条件（如订单类型、库存等级）直接跳过中间步骤，流转到指定的“动作节点”。

### 7.2 层级分工说明
- **规则级 (Rule-level)**: 管理业务大环节之间的接力（如：从“准入校验”交棒给“储位分配”）。
- **阶段级 (Step-level)**: 管理原子算子之间的逻辑编排（如：先查“超收拦截”，再查“温控拦截”）。

### 6.3 步骤类型的概念
所谓的“步骤类型”是一个复合概念，由两个维度定义：
1. **业务主体维度**: 决定了算子“找什么”（库位、库存还是资源）。
2. **执行行为维度**: 
   - **满足即止 (Terminate)**: 寻址成功即返回结果。
   - **累加流转 (Continue)**: 计算结果传递给后置算子。

---

> **专家建议**: 始终在发布前运行 **Shadow Analysis**。即使逻辑看起来完美，非线性逻辑的叠加也可能产生非预期的寻址死角。

## 8. 执行安全约束 (Execution Safety Constraints)

在步骤（Step）层面，系统提供三个执行安全阀，防止候选集爆炸或步骤长时间卡死：

| 参数 | 含义 | 推荐值 |
| :--- | :--- | :--- |
| **最大候选数 (maxCandidates)** | 进入步骤的候选集上限。超出后截断，但仍参与评分排序。 | 500–2000（复杂聚类场景） |
| **步骤超时 (timeoutMs)** | 步骤执行的最长时间。超时后触发 `failoverAction` 兜底。 | 200–800ms（实时链路） |
| **最大输出数 (maxOutputCount)** | 步骤最终输出的候选数上限。常用于限制下一步骤的输入规模。 | 10–50（精选后传递） |

**配置建议：**
- **高并发波次策略**（如：AI 智能波次）：`maxCandidates: 2000, timeoutMs: 800`
- **实时设备调度**（如：叉车交叉任务）：`maxCandidates: 200, timeoutMs: 300`
- 三个参数均为可选，未配置则无限制（仅建议在性能敏感路径上配置）

## 9. 事件触发条件 (Event Trigger Conditions)

策略除了手动激活，还支持**事件驱动自动触发**。在「实例属性 → 事件触发条件」中配置：

| 事件类型 | 业务场景 |
| :--- | :--- |
| `ORDER_ARRIVED` | 订单池积压到阈值时自动触发波次 |
| `INVENTORY_CHANGE` | 库存水位低于设定值时自动触发补货 |
| `WIP_THRESHOLD` | 在途任务数量异常时触发熔断 |
| `EQUIPMENT_STATUS` | 设备状态变更（如：AGV 充电完成）时触发任务分配 |
| `TIME_SCHEDULE` | 固定时间计划触发（如：每天 22:00 执行夜班波次） |
| `MANUAL` | 人工手动触发（默认） |

**冷却时间 (cooldownMs)**：同一触发器在触发后的静默期，防止短时间内重复触发。建议波次策略设置 3–5 分钟，补货策略 2–3 分钟。

---

## 10. 时序/趋势条件算子 (Time-Window & Trend Operators)

除了即时点值比较（如：`库存 < 100`），系统支持对**时间窗口内的聚合值**或**趋势方向**进行判断：

| 算子类型 | 示例 | 业务含义 |
| :--- | :--- | :--- |
| **时间窗口** | 过去 2 小时拣货次数 > 50 | 窗口内的聚合计数/求和/均值 |
| **趋势判断** | 库存消耗速率 TREND_UP | 指标在上升/下降趋势中 |

时序条件需在条件编辑器中将算子切换到「时间窗口」分组，之后展开 `时长 + 单位 + 聚合方式` 三个子字段。

## 11. 资源锁定 (Resource Locking)

**适用场景：** 多条规则并发执行时，防止同一设备/库位/库存批次被重复分配。

### 11.1 锁定类型

| 锁类型 | 行为 | 适用场景 |
| :--- | :--- | :--- |
| **HARD（硬锁）** | 排他性占用。其他步骤尝试获取同一资源时被**阻断**，冲突候选从输出中移除。 | 设备任务分配（同一叉车不能同时被两条规则指派） |
| **SOFT（软锁）** | 可抢占式预留。发生冲突时**产生警告但候选保留**，TTL 超时后自动释放。 | 库存批次预占（同一批次可能被多个波次暂时标记，后续去重） |

### 11.2 配置项说明

| 配置项 | 含义 |
| :--- | :--- |
| 步骤启动时加锁 | 步骤开始执行前即对输出候选占位 |
| 成功后释放 | 步骤正常完成后主动释放（推荐始终开启）|
| Failover 时回滚 | 步骤异常时释放已持锁资源，防止死锁 |
| TTL（仅 SOFT） | 锁的超时时间，超时后无论是否释放都自动解锁 |

### 11.3 策略级回滚策略

在「实例属性」弹窗底部可配置 **步骤失败回滚策略**：
- `NONE`：不回滚（默认，适用于无资源锁步骤）
- `PARTIAL`：仅回滚失败步骤之后才获取的锁（推荐：部分容错）
- `FULL`：释放本次执行中所有已持锁资源（保守策略）

### 11.4 仿真验证

在 Simulator「决策 Trace」标签，**资源锁竞争摘要**卡片展示四个指标：
- **获取**：成功加锁次数
- **释放**：成功释放次数
- **冲突**：锁争用次数（> 0 时红色高亮，说明需要关注资源竞争）
- **回滚**：因 failover 触发的回滚次数

## 12. 成本因子框架 (Cost Optimization Framework)

**业务价值：** 将优化目标从「约束满足 + 排序」升级为「经济最优」——在保证合规的前提下，最小化每次仓库决策的总运营成本。

### 12.1 四维成本建模

系统预置四个成本维度，步骤可按需勾选启用：

| 成本维度 | 计量单位 | 适用场景 |
| :--- | :--- | :--- |
| **人工成本** | ¥/min | 拣货、搬运、整理等人力作业 |
| **设备折旧** | ¥/km | 叉车、AGV 行驶里程折算 |
| **库存持有风险** | ¥/unit/day | 滞销、临期、积压风险 |
| **SLA违约罚款** | ¥/单 | 订单超期或服务等级违约的预估罚款 |

### 12.2 步骤配置

在步骤的「成本优化」折叠区中：

1. **启用成本优化**：开关，关闭时不参与任何成本计算
2. **优化目标**：决定哪个成本维度权重最高
   - `最小化总成本`：所有已选维度综合最优
   - `最小化人工成本` / `最小化设备成本`：专项优化
   - `均衡`：多维度平衡
3. **启用维度**：从四个预置维度中选择本步骤需要纳入计算的项
4. **成本上限（¥）**：单次操作的可接受成本上限。仿真时若超出，Simulator 将高亮警告
5. **成本排序权重（0–100%）**：
   - `0%`：完全由因子得分决定排序（传统行为）
   - `100%`：完全由成本决定排序
   - `30%`：因子得分占 70%，成本优化占 30%（推荐均衡值）

### 12.3 因子与成本的关联

在「因子管理」中，DYNAMIC 因子可以关联一个成本维度（`costMetadata`），系统在计算候选成本时会以因子实际值乘以费率和系数得出贡献成本。

**示例：** `设备空驶距离`（fact-deadhead-dist）× 1.2 ¥/km × 1.5（系数）= 该设备的设备折旧成本贡献。

### 12.4 仿真分析

在 Simulator「决策 Trace」标签，**预估成本分析**卡片展示：
- 本次决策的 **总预估成本（¥）**
- 各维度占比横条（直观看到主要成本驱动因素）
- 每个步骤行上的 **¥ 成本徽章**（定位高成本步骤）

> **专家建议**：成本排序权重建议从 20–30% 开始调试，权重过高会导致因子评分体系失效；运行 Simulator 对比不同权重设置下的候选集变化，找到业务最优平衡点。
