# 收货策略配置抽象方案与实操指南

针对用户提供的多维收货策略表（采购、调拨、直送、退货等），我们通过**“标准化管线 + 参数化配置”**的设计思路进行了高度抽象。

## 1. 核心抽象逻辑：从“一事一议”到“标准插件”

在咱们的系统中，不再为每种收货类型编写死逻辑，而是抽象出两个核心**执行动作 (Steps)**：

### 动作 A：收货准入合规校验 (Compliance Check)
- **作用**：负责“挡掉不合规的操作”。
- **硬性约束 (Filters)**：配置超量、缺量、拣选面限制等。
- **行为类参数 (Params)**：控制是否允许冲销、是否到期自动满单。

### 动作 B：收货储位自动指派 (Allocation)
- **作用**：负责“决定货物放哪”。
- **行为类参数 (Params)**：配置目标储位代码（如 SH, IN, THSH）、是否自动入库。

---

## 2. 典型策略配置案例拆解

通过这种抽象，原本复杂的表格配置在系统中变得清晰简洁：

### 案例 1：采购入库收货策略 (常规严控)
- **准入校验动作**：
    - **硬性约束**：`超量收货 == 禁止超收`，`拣选面限制 == 必须存在`。
    - **节点参数**：`允许缺量收货: TRUE`, `允许冲销: TRUE`, `到期自动满单: FALSE`。
- **储位指派动作**：
    - **节点参数**：`是否自动入库: FALSE`, `指定储位: SH`。

### 案例 2：直送自动入库策略 (极速通道)
- **准入校验动作**：
    - **硬性约束**：同采购。
    - **节点参数**：`是否自动入库: TRUE` (核心区别点)。

### 案例 3：返配/销退策略 (逆向严控)
- **准入校验动作**：
    - **硬性约束**：`缺量收货 == 严禁缺量`。
    - **节点参数**：`允许冲销: FALSE`, `到期自动满单: TRUE` (强制满单)。
- **储位指派动作**：
    - **节点参数**：`指定储位: THSH`。

---

## 3. 方案优势

1.  **极简维护**：如果你想把所有收货类型的“超收”都放开，只需要在拦截动作的参数里点一下，不需要改动每一个策略。
2.  **业务语言对齐**：配置界面直接展示“是否允许超收”、“指定储位代码”等业务术语，实施人员无需理解底层代码。
3.  **高扩展性**：如果未来新增一种“申偿收货”，只需要新建策略并挂载这两个标准动作，调整一下参数（如储位设为 `SCSH`）即可完成配置。
